# mini-motorcad 오프라인 실행 매뉴얼 (Python)

Node.js 없이 **Python만으로** 프로그램을 실행하는 방법입니다. 빌드된 프런트엔드(`app/dist/`)가
저장소에 포함되어 있어, 인터넷·Node 없이 바로 띄울 수 있습니다.

---

## 1. 필요 조건

- **Python 3.8 이상** (표준 라이브러리만 사용 — 추가 설치 불필요)
- 확인: 터미널에서 `python --version`

> FEMM FEA 연동(선택)을 쓸 때만 추가로 FEMM + `pyfemm`이 필요합니다. 핵심 기능
> (형상·파라미터·권선 시각화·효율맵·열해석)은 FEMM 없이 전부 동작합니다.

---

## 2. 가장 간단한 실행

저장소 루트(이 파일이 있는 폴더의 상위)에서:

```bash
python run_offline.py
```

- 자동으로 `http://127.0.0.1:5173/` 가 브라우저에 열립니다.
- 종료: 터미널에서 `Ctrl+C`.

포트를 바꾸려면:

```bash
python run_offline.py --port 8080
```

브라우저 자동 열기를 끄려면 `--no-browser`.

---

## 3. FEMM FEA까지 함께 실행 (선택)

FEMM과 pyfemm이 설치된 PC에서만:

```bash
python run_offline.py --femm
```

- 프런트엔드(5173) + FEMM 서버(8765)가 함께 뜹니다.
- 앱의 **Calculation 탭 → FEMM 검증** 버튼이 이 서버를 호출합니다.
- FEMM 서버만 따로 띄우려면: `python -X utf8 fea/femm_server.py`

---

## 4. (참고) 소스에서 직접 빌드하기

`app/dist/`는 저장소에 포함되어 있어 보통 빌드가 필요 없습니다. 소스를 수정한 뒤
다시 빌드하려면 Node.js가 필요합니다:

```bash
cd app
npm install
npm run build      # → app/dist/ 갱신
cd ..
python run_offline.py
```

---

## 5. 문제 해결

| 증상 | 원인·해결 |
|---|---|
| `빌드물(app/dist/)을 찾을 수 없습니다` | 저장소가 불완전하거나 dist 미포함. 위 4번으로 빌드. |
| `포트 ... 를 열 수 없습니다` | 포트 사용 중. `--port 8080` 등 다른 포트 지정. |
| 브라우저가 안 열림 | 표시된 주소를 수동으로 입력. `--no-browser` 안 줬는지 확인. |
| FEMM 버튼이 응답 없음 | `--femm`으로 실행했는지, FEMM+pyfemm 설치됐는지 확인. |
| 화면이 비어 보임 | 브라우저 캐시 새로고침(Ctrl+F5). 콘솔(F12) 오류 확인. |

---

## 6. 동작 방식 (요약)

- `run_offline.py`는 파이썬 `http.server`로 정적 빌드물(`app/dist/`)을 로컬 서빙합니다.
- React 앱은 단일 페이지 정적 파일이라 별도 백엔드가 필요 없습니다 — 모든 계산이
  브라우저(자바스크립트)에서 실행됩니다.
- FEMM FEA만 예외적으로 파이썬 백엔드(`fea/femm_server.py`)가 필요하며 선택 사항입니다.
- 모두 `127.0.0.1`(로컬)에만 바인딩되어 외부에 노출되지 않습니다.
