## 고친 것 — 회전(ROTATE) 미적용

`extractGeometry`의 회전각 추정(`meanRot`)이 **잔차의 산술평균**이었던 게 원인입니다. 18슬롯 모터는 슬롯 피치가 20°라 잔차 범위가 ±10°인데, **톱니(또는 극)가 좌표축 위에 오도록 그려진 흔한 Motor-CAD DXF**에서는 슬롯 오프셋이 정확히 +10°/−10° 경계에 걸칩니다. 그러면 측정 노이즈에 따라 절반은 +10°, 절반은 −10°로 갈려 **평균이 0으로 상쇄 → 회전이 사실상 적용 안 됨**.

수치로 증명된 차이(노이즈 ±1.5°):

| 실제 오프셋 | 기존 `meanRot` | 수정 `circRot` |
|---|---|---|
| 10° (톱니가 축 위) | **0.04°** ❌ | −9.96° ✅ |
| 9.5° (경계 근처) | **0.65°** ❌ | 9.54° ✅ |
| 6.5° (검증값) | 6.54° ✅ | 6.54° ✅ |

**n중 대칭 원형평균** `atan2(Σsin nθ, Σcos nθ)/n` 으로 교체했습니다(`statorRot`·`rotorRot` 둘 다). [App.jsx](app/src/App.jsx#L243), 진단/검증 스크립트 동일 적용. `verify_autofit.mjs`에 **경계(10°+노이즈) 회귀 시나리오**를 추가해 고정 — 전체 ✅ 통과, vite 빌드도 통과했습니다.

→ **브라우저는 자동 핫리로드** 되니, DXF 다시 불러와 「형상 자동 맞춤」을 눌러 확인해 주세요. 회전이 어긋나면 슬롯·자석도 전부 겹쳐 보이지 않으므로, 이거 하나로 슬롯/자석 정렬이 상당히 맞아 들어갈 수 있습니다.

## 아직 안 고친 것 — 자석 R 면취 / 슬롯 형상

확인해 보니 `runFit`은 이 둘을 **아예 적용하지 않습니다**:
- **자석 R 면취**(`magnetReduction`): 추출/적용 안 함 → 모델은 레퍼런스 기본값 1.3을 그대로 유지. DXF 자석의 면취량과 다르면 안 맞음.
- **슬롯 입구**(`slotOpening`·`toothTipDepth`·`toothTipAngle`): 추출/적용 안 함 → 슬롯 깊이·톱니폭만 맞추고 입구 형상은 그대로.

이 둘은 **실제 DXF의 폴리라인 구조에 따라** 추출식을 짜고 검증해야 합니다(합성 테스트엔 면취·입구가 없어서, 추측으로 짜면 또 안 맞을 위험이 큽니다). 그래서 그 DXF를 직접 보고 맞추겠습니다. 아래 한 줄을 실행해 출력을 붙여 주시거나, 파일 경로만 알려 주세요:

```bash
node tools/diag_dxf.mjs "C:\경로\파일.dxf" -v
```

`-v`가 닫힌 폴리의 반경밴드·각폭 분포까지 찍어줘서, 면취량과 슬롯 입구를 어떻게 뽑을지 바로 설계할 수 있습니다.

## 만든 것 — `.aedt` 설계변수 임포트

Geometry 탭 왼쪽에 **「⬇ AEDT 불러오기」** 버튼(시안색)이 추가됐습니다. `.aedt`를 고르면:
1. `VariableProp` 블록을 파싱하고,
2. 수식·단위를 평가(`D_si=D_ro+2*g`, `asin(...)`, `mm`/`deg`, 변수참조·`sin/cos/sqrt/PI`)해서,
3. 모델 파라미터에 정확히 매핑하고, 적용/미발견 목록을 패널에 표시합니다.
