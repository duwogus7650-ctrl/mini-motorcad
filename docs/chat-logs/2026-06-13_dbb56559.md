### 🤖 Claude · 2026-06-13 00:09:47

검증 완료했습니다. **결론: 핵심 해석 엔진은 README 주장대로 정확하게 짜여 있고 빌드도 정상입니다. 다만 입력 견고성(NaN 전파)과 Tailwind CDN 의존 등 고칠 만한 문제가 몇 개 있습니다.**

## 검증 방법과 결과

**1. 빌드 — 통과.** `npm install` + `vite build` 성공 (번들 190kB, 에러·경고 없음).

**2. Python 검증 스크립트 — 통과.** [validate_engine.py](validation/validate_engine.py) 실행 결과 kw1=0.94521, kw3=0.57735 정확 일치, λ/BEMF/토크는 참조 대비 0.1~0.2% 오차.

**3. JS 엔진 전수 검증 — 통과.** [App.jsx](app/src/App.jsx)의 `compute()` 함수를 그대로 떼어내 Node에서 Motor-CAD FEA 참조값 36개 항목과 비교했습니다. 28개 항목이 1% 미만 오차로 일치(kw1, 토크, 상저항, 동손, 슬롯면적, MLT, 효율, 무부하속도, 스톨전류, 코깅 주기/주파수 등). 3%를 넘는 8개 항목(중량 4건 ~7.5%, Ld/Lq ~8%, Pfe 6.6%, 관성 6.2%)은 **전부 README가 이미 "추정치/오차 5~8%"라고 명시한 항목**이라 문서가 정직합니다. README의 kw5=0.13985 주장도 일치 확인했고, star-of-slots 상배정은 3상 평형(144/144/144)입니다.

**4. 수식 검토 — 이상 없음.** Carter 계수, 쇄교자속(직렬턴 기준), 토크의 병렬회로 처리, 저항 R=ρ·MLT·N/(A·a²), 역률, 스톨, 코깅 LCM 등 표준 수식이고 단위 일관성도 맞습니다.

## 발견된 문제 (심각도 순)

1. **입력 검증 없음 → NaN 전파.** `slotNumber=0`이나 `airgap=0`이면 torque=NaN, `parallelPaths=0`이면 Infinity가 그대로 화면에 표시됩니다. 입력칸([App.jsx:404](app/src/App.jsx#L404))이 `parseFloat(...) || 0`이라 **값을 지우고 다시 타이핑하는 순간 0이 들어가 출력 전체가 NaN으로 변하는** 상황이 실제로 발생합니다. [App.jsx:422](app/src/App.jsx#L422)의 try/catch는 예외만 잡고 NaN은 못 잡습니다.
2. **Tailwind Play CDN** ([index.html:7](app/index.html#L7)). 오프라인이거나 CDN이 막히면 스타일이 전부 깨지고, 프로덕션 사용 비권장 방식입니다. 사내 데이터가 든 사내용 도구라면 devDependency로 설치해 빌드에 포함하는 게 맞습니다.
3. **Geometry 캔버스 리사이즈 버그** ([App.jsx:507](app/src/App.jsx#L507)). 너비 변화만 검사해서 창 높이만 바뀌면 캔버스 해상도가 갱신되지 않아 그림이 왜곡됩니다. SlotViewer 쪽은 매번 재설정해서 문제없습니다.
4. **`onWheel`의 `preventDefault()`** ([App.jsx:593](app/src/App.jsx#L593)). React 17+는 wheel을 passive 리스너로 등록하므로 무시되고 콘솔 경고가 뜹니다. 현재 레이아웃은 페이지 스크롤이 없어 실질 피해는 적습니다.
5. **비정상 슬롯/극 조합 무경고.** 예를 들어 Ns=17 같은 불평형 조합도 조용히 그럴듯한 숫자를 출력합니다. 실수 방지용 경고(슬롯수가 3의 배수인지 등)가 있으면 좋겠습니다.
6. 사소: `exportAll`에서 `URL.revokeObjectURL` 미호출, MaterialsTab 렌더 내부에서 `SteelSel` 컴포넌트 정의(리렌더마다 재생성).

## 구조 평가

단일 파일 1254줄이지만 파서/형상/권선/엔진/UI 섹션 구분이 명확하고, 해석 엔진이 JSX 없는 순수함수로 분리돼 있어 이번 검증에서도 그대로 떼어내 실행할 수 있었습니다. `compute()` + `useMemo` 실시간 재계산 구조도 적절합니다. 엔진부(`compute`, `windingAnalysis`, `parseDxf`)를 별도 모듈로 빼면 자동 테스트를 붙이기 더 쉬워질 겁니다.

수정을 원하시면 1~3번부터 처리하는 것을 권합니다.
