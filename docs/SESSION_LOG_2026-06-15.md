## 1. FEMM 브릿지 (진짜 2D FEA) — 완성·디버깅·검증

웹앱이 형상·권선·전류·진각을 보내면 `fea/femm_server.py`(Flask+pyfemm)가 FEMM 모델을 구성·다위치 해석.

### 핵심 디버깅 (시간순 교훈)
1. COM 오류 → FEMM 4.2 **64bit** 설치 + Flask 워커 `pythoncom.CoInitialize()`.
2. "triangle / small angles" → 보어·로터 완전원 제거, 치페이스/폴갭 아크로 경계 구성.
3. 슬롯·자석 미표시 → `mi_addsegment/mi_addarc`가 끝점 노드 자동생성 안 함 → 모든 점 `mi_addnode` 선행.
4. "problem loading mesh" → 얇은 에어갭에 강제 메시 실패 → **에어갭은 자동메시**.
5. **몇 시간 소요의 진짜 원인 = smartmesh**: 켜지면 블록 메시 크기를 무시하고 자동 미세화(20만 노드).
   `mi_smartmesh(0)` + 형상비례 메시 → **1.2만 노드, 몇 분**.
6. 토크·Ke 둘 다 0 → 벡터퍼텐셜 A∝sin(전기각)이라 자기축 β보다 90° 어긋남 → 정렬·MMF에 90° 보정.
7. 토크 부호 → 전기각 증가방향이 회전방향 반대 → q축 −90°.
8. precision 1e-6 거부("Invalid Precision") → 1e-8 유지.

### 검증 (FEMM ↔ 1250W-jk Maxwell REF, 거친 메시)
| | FEMM | REF | 비고 |
|---|---|---|---|
| Lq | 0.1399 | 0.1401 | 완벽 |
| Ld | 0.1349 | 0.1289 | +4.7% (대칭 ±0.5Ipk 섭동으로 자석 탈포화 과대 해소) |
| Ke | 0.122 | 0.126 | −3% (거친메시) |
| 토크 | 3.61 | 3.73 | −3% (거친메시) |
| 코깅 | — | 9.13 mNm | **신뢰불가**(메시 노이즈 > 코깅) → "참고만" 표시 |

### 설계 포인트
- **형상비례 메시**(치=슬롯피치/6, 자석=두께/3, 코일=슬롯깊이/8, 에어갭=자동) → 모터마다 자동 적응.
- 2D 깊이 = **magneticLength**(스택길이 아님 — 안 그러면 토크 ~7.5% 과대).
- 부하각: 상자기축 ψ_p로 합성 MMF를 d축+90°+진각에 정렬(상순·부호 무관).
- 신규 출력: Ke·무부하 역기전력, 치/요크 자속(Pfe용), Ld/Lq(dq 전류주입+Park투영).
- 전역 락(동시 직렬화), parallelPaths /P, 위치별 진행 로그.

### 완전 FEMM 기반 보정
"FEMM 보정 적용" → `compute(G,W,M,C,cal)` 에 **λ·kT(포화토크)·Pfe·Ld·Lq** 주입 →
토크·EMF·출력·전압·역률·효율·T-N·효율맵 전부 FEMM 기반. **효율맵은 점마다 FEMM 안 돌림**(보정된 빠른 엔진).
형상/권선/재질 변경 시 자동 해제. 거친메시 FEMM은 토크를 수% 낮게 보므로 **REF 있는 모터는 해석식이 더 정확**,
FEMM 보정은 **레퍼런스 없는 새 모터**에 유용.

## 2. 전체 정밀 감사 (88건 발견 → 58건 확정) — 주요 수정

- **포커스 풀림**: Box/SteelSel을 컴포넌트 내부 정의 → 모듈 레벨(리마운트 제거).
- **역률/단자전압**: 진각 무시 → 정식 dq식(id=−Ipk·sinγ).
- **NaN 전파**: `pp=round(Ke/λ)`(0/0) → `out.pp` 직접. KtLine·Tshaft·byDepth 가드.
- **DXF**: ARC를 큰 각도만 동심원 후보(H2), LINE 끝점 가드, median 짝수 평균, 내보내기 NaN 제외.
- **권선**: throw 무효·beltMap 60°경계 epsilon·3상 균형검사(경고).
- **효율맵**: 약계자 시 d축 자속(λ+Ld·id) 감소를 철손에 반영.
- **자석 질량**: 실제 면취 단면(shoelace) → 형상·질량 일관.
- **열 시정수**: 정상상태 일관 등가저항 Req=(hot−Ta)/Qtot.

## 3. Self-Check 탭 (자동 검증 — 빨강/초록)

"내가 못 잡아도 도구가 먼저 빨개진다"는 사용자 우려에 대한 답.
- **참조 대조**(초록<2%/주황<8%/빨강≥8%) · **물리 타당성**(포화·전류밀도·점적률·전력수지·NaN) · **FEA 교차검증**.
- 실제 적발 사례:
  1. By −22.9% → 형상 드리프트(외경 115.x, DXF추출 잔재) → Geometry "기준형상 리셋(1250W)" 버튼으로 복원.
  2. Pfe +6.6% → 철손계수 kh/ke가 By버그에 가려 높게 맞춰져 있던 것 → ×0.938 재정합(효율도 REF 95.21에 근접).
- 철손계수 갱신: 20PNX1200F **kh=0.0212, ke=4.157e-5**.

## 4. 기타

- **DXF 형상 자동 맞춤**: 불러온 DXF를 파라메트릭 모델에 겹치도록 **중심·스케일·회전**까지 자동 정렬
  (`runFit`). 슬롯 무게중심각으로 `statorRot` 추정→`rot=−statorRot`, 중심을 원점으로 이동, m↔mm 단위 적용,
  자석은 `rotorRot=wrap(ex.rotorRot−ex.statorRot, 극피치)`로 맞춤. 기존 "형상 추출만"(치수 적용·정렬 X)과 별도 버튼.
  검증: `tools/verify_autofit.mjs` — 알려진 중심·단위·회전의 합성 18s/16p 모터에 실제 추출+변환 적용 →
  **중심오차 0, 슬롯·자석 각도잔차 5.7e-14°(기계정밀도)**, 슬롯반경·OD/보어/샤프트·슬롯/극수 전부 정확.
- **CSV 추출**: `motor_spec.csv`(사양표) · `tn_curve.csv`(T-N), UTF-8 BOM(Excel 호환).
- **다크 컨트롤패널 UI**: 네이비+시안, 브래킷코너 패널(14px 라운드+그라데이션), 테크니컬 폰트, 캔버스 다크화.
- **완전 오프라인**: Tailwind·폰트 CDN→로컬 번들(PostCSS + @fontsource). 인터넷 없이 구동(최초 npm install만 온라인).
- **앱 모드 실행**: run_gui.bat가 Edge/Chrome `--app` 독립 창(주소창·탭 없음, Win11 둥근 모서리).
  PowerShell로 경로 따옴표 안전 처리, 중복 서버(5173/8765) 자동 감지·재사용. **run_femm_server.bat** 단독 추가.
